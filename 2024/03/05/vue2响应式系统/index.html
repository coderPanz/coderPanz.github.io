介绍

Vue 最独特的特性之一，是其非侵入性的响应式系统。数据模型仅仅是普通的 JavaScript 对象。而当你修改它们时，视图会进行更新。

追踪变化

把一个普通的 JavaScript 对象传入 Vue 实例作为 data 选项，Vue 将遍历此对象所有的 property，并使用 Object.defineProperty 把这些 property 全部转为 getter/setter。(getter 和 setter 是属性访问器（accessors），它们允许你在访问或设置对象的属性时执行自定义的操作。)访问这个属性或更新时触发对应操作。

这些 getter/setter 对用户来说是不可见的，但是在内部它们让 Vue 能够追踪依赖，在 property 被访问和修改时通知变更。

每个组件实例都对应一个 watcher 实例(观察者)，它会在组件渲染的过程(渲染过程需要访问数据触发getter)中把“接触”过的数据 property 记录为依赖。之后当依赖项的 setter 触发时，会通知 watcher，从而使它关联的组件重新渲染。这就是vue2响应式的机制。

缺陷及解决方法

对象

Vue 无法检测 property 的添加或移除。由于 Vue 会在初始化实例时对 property 执行 getter/setter 转化，所以 property 必须在 data 对象上存在才能让 Vue 将它转换为响应式的。并且使用的 Object.defineProperty 中的getter和setter只能监听到访问属性和属性更新操作。

对于已经创建的实例，Vue 不允许添加响应式的property属性。在Vue中提供了 Vue.set(object, propertyName, value) 方法向嵌套对象添加响应式 property。或者使用Vue实例方法 vm.$set(object, propertyName, value)。

数组

Vue不能检测以下情况的操作

  • 利用索引进行赋值操作：vm.arr[index] = newValue

  • 修改数组长度： item.arr[indexOf] = newValue，数组修改长度后会自动截取对应个数的项。

索引赋值操作响应式更新方法

声明响应式 property

Vue 不允许动态添加根级响应式 property，必须在初始化实例前声明所有根级响应式 property。

队列的异步更新

Vue 在更新 DOM 时是异步执行的。只要侦听到数据变化，Vue 将开启一个队列，并缓冲在同一事件循环中发生的所有数据变更。如果同一个 watcher 被多次触发，只会被推入到队列中一次。缓存避免了不必要的计算和对DOM的操作，当你设置 vm.someData = 'new value'，该组件不会立即重新渲染。当刷新队列时，组件会在下一个事件循环“tick”中更新。但是如果你想基于更新后的 DOM 状态来做点什么，为了在数据变化之后等待 Vue 完成更新 DOM，可以在数据变化之后立即使用 Vue.nextTick(callback)。这样回调函数将在 DOM 更新完成后被调用。

在组件内可以使用 vm.$nextTick() 实例方法，且回调函数中的 this 将自动绑定到当前的 Vue 实例上。
